# Submission Runbook

Use this checklist to move `X Layer Execution Proof Kit` from repo-ready to submission-ready.

## 1. Repo

- confirm `main` is pushed to the public GitHub repo
- confirm README includes:
  - project intro
  - architecture overview
  - usage surface
  - OnchainOS / Uniswap usage
  - working mechanics
  - team
  - X Layer ecosystem positioning
  - onchain identity

## 2. Live Proof

- keep the real example attempts and rendered proofs in `examples/`
- capture one CLI screenshot rendering:
  - a successful live proof
  - a failed live proof
- highlight that the examples came from live X Layer systems, not only mocks

## 3. Public Posts

- post the Moltbook submission using [submission-post.md](/Users/yanqing/Documents/GitHub/miraix-interface/projects/xlayer-execution-proof-kit/docs/submission-post.md)
- publish an X post with:
  - repo link
  - one screenshot of a rendered proof bundle
  - `#onchainos`
  - `@XLayerOfficial`

## 4. Submission Form

Prepare these links:

- GitHub repo
- README
- live examples doc
- Moltbook post
- X post
- demo video

## 5. Demo Video

Recommended 60-second structure:

1. problem: agents say they traded but rarely prove what happened
2. input: show one execution attempt JSON
3. output: show one rendered proof bundle
4. failure path: show one live execution-error example
5. positioning: reusable evidence layer for X Layer agents
